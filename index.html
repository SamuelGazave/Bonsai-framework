Bonsaï

A framework for e-mail design

View on GitHub Download .zip Download .tar.gz

Hi i'm Samuel.

In my curent job i build a lot of e-mail campaigns and as html email can be tricky, i built a tool to help me create solid templates and newsletters very fast. It comes with a grid system and pre-built elements i use the most in my creations in order to help graphic designers to easily create mobile friendly designs.

strengths

  1. The grid Helps graphic designers to understand how heir creation will behave on mobile devices. Coders can also use it to quickly slice the PSD files graphic designers gives them.
    As i cann see most of the time, those who create e-mails on photoshop are not allways those who code them (most of the designers who sends me PSDs design for print but have very few web design understanding).
    This tool helps everyone working together to get a better result on responsive e-mail design.

  2. The HTML is separate from the CSS. It allows me to easaly maintain a graphic chart on a large collection of templates linked to the CSS file.
    For example, when changing from winter to summer collection, i just have to change colors or type styles into the CSS file and my entire collection is up-to-date.
    This is also a good way to have a typographic constancy within my e-mail collection as i set the differnt styles in the CSS before apply the correpondant classes in my HTML.

  3. Using an inliner to mount e-mails is a good way to avoid errors when coding. It allows me to manage links, alt or title tags in a HTML file free from any style and when it's done, the inliner gets all my css inline for me. I can't forget anything and my e-mail is good to go.

Weaknesses

When an e-mail is broken, it usualy is because of theses points :

  1. Outlook conditionnal comments width don't match the actual size of the columns. I have to pay attention to this each time i have to work on an e-mail but when i forget, the error is easy to find and repair.

  2. The size in the whidth tag of an image don't match the size it must have in a column with padding applied arround it. it causes Outlook to display the image shorter or wider than it should be. It is also an error easy to find and repair.
